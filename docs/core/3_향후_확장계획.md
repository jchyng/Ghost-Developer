# AI Company - 향후 확장 계획 (v2.0+)

> MVP(단일 핵심 루프)가 안정적으로 동작한 이후, 실제 사용하며 필요성이 느껴질 때 점진적으로 추가할 고급 기능들의 목록입니다.

## 1. Telegram 기반 원격 상호작용 (Remote Handoff)
*   **배경:** 자리를 비웠을 때 AI가 작업하다가 막혔는지 확인하고 싶을 때.
*   **기능:** 
    *   작업 완료, 에러 종료 등 주요 상태 변화 시 Telegram 알림.
    *   **GPT-Proxy 도입:** AI가 판단 불가능한 상태(예: 입력 프롬프트 대기 상태로 N분 경과)가 되면, GPT가 마지막 화면의 상황을 요약하여 Telegram으로 질문 전송.
    *   **Inline Keyboard:** Telegram에서 타이핑할 필요 없이, 객관식 버튼으로 AI에게 다음 행동을 지시.

## 2. 작업 이력 및 비용 영구 보존 (SQLite)
*   **배경:** 에이전트를 오래 쓰다 보니 과거에 어떤 프롬프트로 어떤 결과가 나왔는지, 토큰 비용이 얼마나 들었는지 트래킹이 필요할 때.
*   **기능:**
    *   In-memory 큐를 SQLite로 마이그레이션.
    *   작업 성공/실패율, 소요 시간, 예상 소모 비용(`--max-budget-usd` 활용) 대시보드 구축.

## 3. 스마트 컨텍스트 스트리핑 (ANSI 제거)
*   **배경:** GPT-Proxy가 상황을 판단하기 위해 터미널 출력을 읽어야 하는데, xterm용 ANSI 코드가 섞여 있어 토큰이 낭비되고 분석률이 떨어질 때.
*   **기능:**
    *   백엔드 스트림 스플리터 구현. 
    *   웹소켓으로는 Raw 데이터(ANSI 포함)를 보내고, 내부 분석용 버퍼에는 정규식으로 색상/커서 제어 문자를 제거한 Plain Text만 저장.

## 4. 커스텀 Circuit Breaker
*   **배경:** AI가 특정 파일에서 계속 같은 에러를 내며 무한 루프에 빠져서 비용만 날리는 현상이 자주 발생할 때.
*   **기능:**
    *   동일한 에러 메시지가 3회 이상 반복 출력되거나, 소스코드 변경 없이 단위 테스트만 계속 실패하는 패턴을 감지.
    *   자동으로 프로세스를 일시 정지(Pause)시키고 사용자 개입(Manual Intervene)을 요청.

## 5. 멀티 유저 & 인증 레이어
*   **배경:** 이 도구를 팀원이나 다른 사람과 공유하고 싶어질 때.
*   **기능:**
    *   웹소켓 및 API 엔드포인트에 토큰(JWT) 기반 인증 추가.
    *   사용자별 작업 공간(Workspace) 격리.
