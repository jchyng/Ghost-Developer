# Core Principles (핵심 개발 철학)

> "조금 부족하더라도 핵심 기능 위주로 빠르게 만들고, 내가 직접 사용해보면서 필요한 것만 조금씩 개선한다."

이 문서는 Ghost Developer 프로젝트를 진행하는 내내 모든 의사결정의 기준이 되는 절대 원칙입니다.

## 1. YAGNI (You Aren't Gonna Need It)
- **과설계 절대 금지:** '나중에 필요하지 않을까?'라는 생각으로 미리 만들지 않는다.
- **오직 1인(나)을 위한 도구:** 다중 사용자, 권한 관리, 엔터프라이즈급 안정성(예: DB를 활용한 복잡한 큐 관리)은 고려하지 않는다.
- **최소 기능 출시(MVP):** 가장 단순한 형태의 In-memory 큐와 웹 터미널 연동만으로 첫 버전을 완성한다.

## 2. Thin Wrapper (브리지 레이어는 거들 뿐)
- **Claude Code 순정 기능 극대화:** AI의 페르소나, 룰, 서브에이전트 스폰 등은 모두 `CLAUDE.md`와 Claude Code의 네이티브 기능에 전적으로 위임한다.
- **지능은 AI에게, 서버는 파이프만:** Python 서버는 WebSocket 중계, PTY 관리, DB 저장 등 인프라만 담당한다.
- **v2 예외: Orchestrator는 지능을 가진다.** gpt-5.4-mini 기반 Orchestrator는 사용자 의도를 해석하고 Claude Code 호출 전략을 결정한다. 단, Orchestrator의 지능도 "Claude Code를 어떻게 쓸지 판단"에 한정되며, 직접 코딩을 하지 않는다.
- **안 믿을 거면 안 쓴다:** `dangerously-skip-permissions`를 켜고 Claude Code의 판단을 100% 신뢰한다.

## 3. 실용주의적 안전장치
- 복잡한 에러 분석(Circuit Breaker)이나 규칙 기반(Rule-based) 출력 필터링을 직접 구현하지 않는다.
- **수습 가능성 보장:** 복잡한 방어 로직 대신, 작업을 시작하기 전후에 기계적으로 `git commit`을 남겨 언제든 롤백할 수 있게 만든다.

## 4. 진화하는 시스템
- 처음부터 메신저(Telegram) 연동, 화려한 대시보드를 만들지 않는다.
- PTY 연동이 되고, 내가 웹에서 명령을 내려서 Claude가 코딩을 완료하는 **핵심 루프(Core Loop)** 하나가 완벽히 동작하는지부터 확인한다.
- 사용하면서 진짜로 불편한 점이 생겼을 때만 해당 기능을 추가한다.
